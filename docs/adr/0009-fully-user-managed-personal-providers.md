# 个人供应商由用户完全自主管理

个人供应商不受供应商目录、管理员审核或管理员模型验证约束。用户可以自由填写
供应商名称、Base URL、API Key、Model ID 等个人配置，并自行启用、停用和使用；
配置与生成结果仅归本人管理。

服务器不限制个人供应商 Base URL 的协议或目标网络范围，包括 HTTP、HTTPS、
公网、内网、localhost、链路本地地址及重定向目标。该自由度意味着登录用户可以
借助服务器网络访问其可达地址，产品不提供 SSRF、内网探测或云元数据地址拦截。
这是在内部用户可信前提下接受的明确风险。

地址开放不等于请求协议开放。个人用户必须从系统已经实现并测试的调用协议中
选择，例如 OpenAI Images、OpenAI Responses 或火山方舟 Ark。首个交付阶段不
支持自定义请求模板、任意 Header 或自由 JSON 请求体。

个人用户为模型选择系统内置能力档案；未知模型使用“通用基础”档案。个人用户
不能自行声明尺寸、参考图上限、流式输出等能力或填写自由 JSON 能力参数。该规则
是产品请求契约，不是管理员审核；供应商名称、地址、密钥和 Model ID 仍完全自主。

部门供应商仍由管理员通过供应商目录配置并使用内置模型能力档案。管理员保存并
启用模型后，团队成员即可使用；真实生图验证结果作为管理员诊断信息，不阻止模型
进入工作区。本决策不改变部门供应商的审核、额度和审计规则。

## Considered Options

- 个人供应商也必须从管理员目录选择：服务器边界清晰，但个人用户无法自行接入
  新供应商或私有接口。
- 允许任意公网 HTTPS，但拦截内网和危险地址：可以降低 SSRF 风险，但仍不满足
  个人配置完全开放的要求。
- 个人供应商完全开放、不做地址控制：个人用户拥有最大接入自由，同时服务器接受
  其网络访问风险。本仓库选择此方案。

## Consequences

- 个人供应商的创建、修改、验证、启停和模型维护不需要管理员审批。
- 个人供应商调用必须经过系统内置协议适配器；自由填写的 Base URL 不获得自定义
  请求结构、任意 Header 或执行代码的能力。
- 个人模型必须引用系统内置能力档案；未知模型只能使用通用基础能力。
- 管理员不能查看个人 API Key，也不能使用个人供应商代用户生成；现有密钥加密、
  所有者隔离和任务审计要求继续生效。
- 服务端不得把个人供应商的任意地址错误套用为部门供应商或提供给其他用户。
- 个人供应商请求仍需记录所有者、目标主机、模型、时间和结果状态，但审计记录
  不得包含 API Key 或完整敏感请求内容。
- 部署方必须自行承担内部可信用户可以访问服务器可达网络地址的安全后果。
