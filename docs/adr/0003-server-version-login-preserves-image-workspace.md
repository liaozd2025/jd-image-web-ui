# 服务器版本要求登录并保留原图片工作区

本产品是部署在服务器上、由用户通过浏览器访问的服务器版本。使用 Web UI 前
必须通过管理员创建的用户名和密码登录。登录只是原图片工作区之前的访问要求，
不能另建服务器首页或简化生图表单来替代原工作区；账号、会话、数据归属和权限
作为外围入口与后端能力接入。登录成功后，用户继续使用原有的图片生成、编辑、
素材、提示词、任务和结果操作体验。

## Considered Options

- 重新创建登录后的工作台：账号管理概念更容易集中展示，但会丢失原图片
  工作区的完整能力和用户习惯。
- 在原图片工作区前增加登录，并把用户归属适配到既有功能：产品体验连续，登录
  改造也不会演变成重做图片产品。本仓库选择此方案。
- 同时保留免登录访问和登录访问：会产生两套身份和数据语义，也不符合所有用户
  必须先由管理员创建并登录的使用规则，因此不采用。

## Consequences

- 恢复并复用原图片工作区的页面结构、交互和完整功能，当前新建的服务器管理页
  不再作为普通用户的产品首页。
- 所有访问者必须登录，系统不保留免登录个人模式；普通用户登录后直接进入原图片
  工作区。
- 普通用户所需的账号和会话入口嵌入现有导航、头像或设置区域；管理员能力使用
  独立入口，不挤占图片操作主流程。
- 普通用户的任务、生成结果、上传素材和其他个人内容彼此隔离；管理员可以跨用户
  查看这些内容。
- 任何新界面生成、明显布局调整或核心交互变化，必须先说明范围并获得用户明确
  确认后才能实施。
