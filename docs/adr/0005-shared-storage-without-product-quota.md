# 共享存储不设置产品级配额

共享图库、共享模板和共享提示词片段统一存放在部门服务器硬盘中，不再由管理员
设置共享空间额度，也不因产品额度拒绝新增共享资产。系统继续统计共享资产占用
空间，真实磁盘容量与告警交由服务器运维保障；单个用户的个人存储额度保持不变。
这样可以去掉对部门共享资源没有实际价值的额度维护，同时保留容量观测能力。

## Considered Options

- 保留管理员可配置额度：可以提前阻止共享空间增长，但会增加不必要的日常维护，
  且当前服务器硬盘不是需要在产品内分配的稀缺资源。
- 设置一个很高的固定额度：表面上减少触发概率，但仍保留无业务意义的隐藏上限，
  未来仍可能意外阻断上传。
- 取消产品级额度，仅保留用量统计和运维监控：共享资产不会被任意业务上限阻断，
  磁盘容量问题回到服务器运维层处理。本仓库选择此方案。

## Consequences

- “共享资产与存储”只展示已用空间和资产统计，不提供共享空间额度设置。
- 共享资产写入不再执行共享额度校验；磁盘不可写或空间耗尽时仍必须明确失败，
  不得产生只有数据库记录或只有残留文件的半成品。
- 用户个人存储额度、部门供应商调用额度和按用户调用额度不受本决策影响。
