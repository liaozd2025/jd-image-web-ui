# 生图模型由用户逐任务显式选择

一个个人供应商或部门供应商可以在同一供应商版本中配置多个生图模型，共享同一
套供应商凭据与接口地址。部门供应商由管理员、个人供应商由所有者指定默认模型；
用户在图片工作区每次生成时可以手动选择模型，系统按用户与供应商记住最近一次
选择。当该模型停用或不再可用时，系统回到对应供应商默认模型。供应商调用失败
时不得自动改用其他模型。

每个任务必须记录实际选择的生图模型和供应商版本。模型之间的输出数量、分辨率、
参考图限制和编辑方式等差异由模型能力档案表达，前端只开放所选模型支持的设置，
服务端执行同样的能力校验。

## Considered Options

- 每个模型创建一个独立供应商：实现简单，但会重复维护 API Key、接口地址、额度
  和权限，并把模型选择错误地表达成供应商切换。
- 只允许管理员设置一个全局模型：配置简单，但用户无法按任务在通用生成与精准
  编辑等能力之间选择。
- 调用失败后自动切换模型：可能提高表面成功率，但会在用户不知情时改变能力、
  输出语义和费用，历史任务也难以复现。
- 一个供应商配置多个模型，由用户逐任务显式选择：复用凭据和额度，同时保持
  任务行为可预期、可审计、可复现。本仓库选择此方案。

## Consequences

- 供应商配置必须支持多个生图模型，并且只能有一个配置默认模型。
- 图片工作区必须展示当前模型并允许用户在可用模型中切换。模型选择器放在提示词
  卡片底部操作行右侧，即“清空/查找”右边、右侧“管理模板库”按钮左边；不放在
  输出设置或顶部导航，也不得替换或重排原有图片工作区主体。
- 模型选择器使用紧凑下拉框：收起时显示当前模型名称；展开后显示模型名称与一行
  能力摘要，供应商默认模型带“默认”标识。能力摘要只展示能力档案中的稳定差异，
  例如“连续组图 · 最高 4K”或“精准编辑 · 最高 2K”。
- 提示词卡片中的模型位置始终保留：有多个可用模型时可以展开选择；只有一个时
  显示模型名称但不可展开；没有可用模型时显示“未配置生图模型”，禁止提交生成
  并提供进入系统设置的引导。
- 个人/部门供应商仍在现有输出设置中选择。提示词卡片的模型下拉只展示当前供应商
  下的可用模型，不把多个供应商的模型合并；切换供应商后刷新模型列表，并恢复
  该用户在该供应商下最近选择的模型。
- 切换模型时保留当前提示词和全部参考图。尺寸、输出数量等模型相关参数按模型
  分别记住最近一次有效值，首次选择某模型时使用其能力档案默认值；因切换而调整
  参数时必须给出可见提示，不得静默修改。
- 参考图数量超过目标模型限制时保留全部参考图并标明超限，同时禁止提交生成；
  系统不得自动删除、截断或重新排序用户选择的参考图。
- 系统记住每位用户在每个供应商下最近选择的模型；模型停用时回到默认模型。
- 重试和历史任务复用必须保留原模型；原模型不可用时要求用户重新选择，不得
  静默改用其他模型。
- 任务历史卡片直接显示生图模型名称标签；任务详情显示个人/部门供应商作用域、
  Model ID、能力档案、Seed、Prompt 优化和实际输出格式，支持用户判断与复现。
- 任务历史和详情不得显示 API Key 或完整 Base URL；个人供应商地址只保留必要的
  脱敏主机信息用于所有者识别和审计。
- 调用失败只报告所选模型的失败，不触发模型或供应商自动降级。
