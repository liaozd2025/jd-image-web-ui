# 生图高级参数由用户显式选择并写入任务

会改变模型输入或结果表达的生图高级参数必须由用户显式选择，并写入任务快照。
界面只显示当前模型能力档案支持的参数，服务端使用同一档案校验，不向普通用户
提供自由 JSON 参数入口。

Prompt 优化默认关闭，避免在用户不知情时改写提示词。用户可以主动开启：
Seedream 5.0 Lite 提供标准优化，Seedream 5.0 Pro 提供标准优化和快速优化；
具体可用模式以对应模型能力档案为准。

Seed 默认使用随机模式，但服务器必须在提交任务时生成并保存一个供应商接受的
具体值，不能只保存“随机”。用户可以切换到固定模式并填写 Seed。历史任务重试
沿用原 Seed；界面只承诺它可以提高结果一致性，不承诺像素级复现。

本产品固定不启用供应商水印。WebUI 不提供水印开关；模型接口支持显式关闭时，
服务端必须请求关闭。系统也不对生成结果额外叠加产品水印。

Seedream 模型默认输出 PNG，用户可以按任务选择 JPEG；不支持的 WebP 不在界面
中出现。系统按模型记住用户最近一次有效输出格式。

首期“生成数量 1–4”统一表示独立多结果。对于单次请求只能输出一张的模型，
服务端按数量发起多次独立调用；界面必须标明“独立结果”。Lite 连续组图是第二
阶段的独立模式，不得复用“生成数量”冒充。

独立多结果保存一个任务基础 Seed，第一个结果使用基础值，后续结果按顺序使用
基础值加结果序号，例如四张使用 `Seed`、`Seed+1`、`Seed+2`、`Seed+3`。服务端
按模型允许范围处理边界，并记录每张结果实际提交的 Seed。

独立多结果中只有部分调用成功时，任务标记为部分完成并保留成功图片。用户可以
补生成失败结果；系统只重试失败序号和这些序号原本对应的 Seed，不删除或重新
调用已成功序号。

当生成数量大于一时，“开始生成”附近直接显示“将发起 N 次独立调用”。使用部门
供应商时同时显示预计额度消耗；个人供应商只显示调用次数。该提示为行内信息，
不增加二次确认弹窗。

## Considered Options

### Prompt 优化

- 默认启用 Prompt 优化：新手可能更容易得到完整提示词，但会隐式改变用户输入，
  也让同一提示词在不同模型上的比较失真。
- 完全不开放 Prompt 优化：行为最简单，但无法使用供应商提供的有效增强能力。
- 默认关闭、由用户按任务显式开启：保留原始提示词控制权，同时让需要的用户
  可以使用模型支持的优化模式。本仓库选择此方案。

### Seed

- 省略 Seed 并让供应商随机生成：提交简单，但任务历史无法保存用于近似复现的
  实际值。
- 随机模式生成具体 Seed，固定模式接受用户输入：既保留日常随机结果，也让任务
  可以重试和比较。本仓库选择此方案。

### 水印

- 默认关闭但允许用户开启：保留供应商能力，但会增加一个当前没有业务需求的
  设置，并可能让用户误生成带标识的正式素材。
- 固定关闭且不展示入口：生成结果保持干净，界面也更简单。本仓库选择此方案。

### 输出格式

- 所有供应商共用同一格式列表：界面一致，但会显示模型不支持的格式并在提交后
  才失败。
- 由模型能力档案限制格式，Seedream 默认 PNG、可选 JPEG：界面只展示真实可用
  选项，并保留用户对文件体积和保真度的选择。本仓库选择此方案。

### 多张输出

- 严格限制为供应商单次调用的输出数量：协议最直接，但会让 Pro 无法使用现有
  多候选工作流。
- “生成数量”表示多次独立调用：保留 1–4 张候选结果，并与模型原生连续组图
  明确区分。本仓库选择此方案。

## Consequences

- 提示词卡片红框位置只放生图模型选择器。Prompt 优化、Seed、输出格式等参数继续
  放在现有“输出设置”的高级区域，并随当前模型能力档案动态显示或隐藏。
- Prompt 优化入口位于高级设置，默认状态为关闭。
- 切换模型后，不支持的优化模式必须回到关闭或该模型最近一次有效值，并明确提示。
- 任务历史与重试必须保留实际 Prompt 优化模式，不得根据当前界面默认值重新推断。
- 服务端不得把界面未展示或能力档案不支持的 Prompt 优化参数发送给供应商。
- 任务快照必须保存具体 Seed 及其来源模式；固定 Seed 必须按模型能力档案校验范围。
- 历史任务重试沿用原 Seed；如果目标模型不再支持 Seed，必须要求用户重新确认，
  不得静默丢弃该参数。
- 独立多结果的任务保存基础 Seed，每张图片保存实际 Seed；重试同一任务时继续
  使用原序列，不得重新随机生成。
- 独立多结果必须支持部分完成状态；成功结果立即可用，补生成只创建失败序号的
  新任务尝试，并保留与原任务的关联。
- 独立调用次数和部门预计额度必须在提交前可见；个人供应商没有价格元数据时不得
  展示猜测金额或虚假成本估算。
- 能力档案和请求映射不得向图片工作区开放供应商水印参数。
- 供应商支持关闭水印时，任务请求必须使用关闭状态；系统不得对输出文件执行
  额外水印处理。
- 输出格式选项来自模型能力档案；切换模型时恢复该模型最近一次有效格式，首次
  使用 Seedream 时选择 PNG。
- 任务快照、供应商请求、结果文件扩展名和 MIME 类型必须使用同一实际输出格式。
- 独立多结果必须记录目标数量、各次实际调用和各自结果，不能伪装成供应商一次
  返回多张。
- 连续组图在第二阶段使用独立任务参数和界面文案，不得改变历史“生成数量”的语义。
