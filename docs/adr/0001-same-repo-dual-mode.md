# 同仓库双模式：单机模式与服务器多用户模式共用一套代码

部门需要把本工具部署为多用户服务（内网 ≤30 人，登录访问、数据按用户隔离），
而单机/桌面形态仍要继续发展。我们决定在同一个仓库里把"服务器模式"做成与
"单机模式"并列的部署形态，而不是 fork 一个服务器专版，也不是每用户跑一个
实例靠反代做登录。理由：作者长期单人维护，fork 意味着每个新功能（如新供应商
适配）都要双份移植；多实例方案则无法承载账号体系、共享资产和部门供应商这些
应用内概念。代价是存储、设置、队列等模块都要同时兼容"单份数据"和"按用户
分区"两种布局，新功能开发时必须考虑两种模式。

## Consequences

- 服务器模式下 Codex / ChatGPT OAuth 通道完全禁用：该通道复用个人 ChatGPT
  登录态，多用户共用等于全部门冒用一个人的账号身份，风险主体是部署者。
  单机模式不受影响。
- 存储继续用 SQLite + 文件系统（按 `data/users/<用户>/` 分区），不引入
  Postgres/对象存储——规模上限（≤30 人内网）是这个选择的前提，若规模假设
  改变需重新评估。
