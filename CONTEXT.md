# CONTEXT

领域词汇表。只记录术语与含义，不记录实现细节。

## 术语

### 单机模式（Local Mode）
现有形态：单人本机运行的 WebUI 工作台（含桌面打包与 CLI），无登录，
数据单份。多用户改造后此形态继续存在，不受影响。

### 服务器模式（Server Mode）
部署在部门服务器上的多用户形态：需登录访问，数据按用户隔离，
Codex OAuth 通道不可用（仅 OpenAI 兼容 API）。与单机模式共用同一代码库。

### 用户（User）
服务器多用户部署中，通过用户名 + 密码登录系统的个体。账号由管理员创建，
用户不能自助注册。用户可以修改自己的密码。每个用户拥有各自独立的
API 供应商配置（含 API Key）和图片内容。

### 个人图库（Personal Gallery）
每个用户私有的参考图素材库，上传素材默认进入这里。单机版中原名
"公用图库"——该旧名在多用户语境下有歧义（会被误解为跨用户共享），
故废弃："公用/共享"一词今后专指跨用户资源。

### 共享图库（Shared Gallery）
全部门可见的参考图素材库，存放品牌素材、产品图等公共资源。
所有用户可读、可上传；删除仅限上传者本人与管理员。

### 个人资产 / 共享资产（Personal / Shared Assets）
统一的归属规则：可复用资产——图库素材、提示词模板、提示词片段——
都分为个人层（各自私有）与共享层（全部门可读，人人可投稿，删除限
投稿人与管理员）。生成结果（任务与图片）只属于个人层，不进入共享层。

### 个人供应商（Personal Provider）
用户自己配置的 API 供应商（Base URL + API Key + 模型 + 调用方式），
仅本人可见可用，费用与额度由本人承担。

### 部门供应商（Shared Provider）
管理员配置的 API 供应商，全部门用户可选用；API Key 对普通用户不可见。
适用于部门统一付费的账号。上游额度与并发是全部门合用的。

### 管理员（Administrator）
拥有管理权限的特殊用户：创建用户账号、重置任何用户的密码、停用账号，
可在界面中查看所有用户的图片与任务历史（部门工具定位，内容属工作产出，
隔离的目的是"各自管理"而非"互相保密"），并可查看部门供应商的按用户
用量汇总。

### 停用（Deactivation）
账号生命周期的唯一下线方式：停用后禁止登录，数据原地保留、管理员仍可
查看。系统不提供删除账号的功能；物理清理属于运维操作，不属于产品语义。
