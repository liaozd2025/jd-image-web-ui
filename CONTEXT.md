# CONTEXT

领域词汇表。只记录术语与含义，不记录实现细节。

## 术语

### 图片工作区（Image Workspace）
用户进行图片生成、图片编辑、参考图选择、提示词编排、参数设置、任务查看和
结果管理的核心操作界面。原版工作区是产品交互基线，用户登录改造不得以新的管理
首页或简化生图表单替代它。
_避免使用：服务器首页、任务提交页、管理工作台_

### 用户密码登录（Username and Password Sign-in）
使用 Web UI 前的必经步骤。账号由管理员创建，用户必须输入用户名和密码登录，
登录成功后进入原图片工作区；系统不允许匿名访问，也不开放自助注册。它只是
Web UI 的访问要求，不是一种新产品形态或一套新工作区。
_避免使用：单机团队能力、SaaS 模式、新版工作区_

### 服务器版本（Server Version）
程序部署在服务器上，用户通过浏览器访问的当前产品版本。服务器版本要求用户
先使用管理员创建的用户名和密码登录，登录后进入原图片工作区。
_避免使用：单机团队版、双模式_

### 用户（User）
通过用户名 + 密码登录系统的个体。账号由管理员创建，用户不能自助注册。
用户可以修改自己的密码。每个用户拥有各自独立的 API 供应商配置
（含 API Key）和图片内容；普通用户之间不可查看彼此内容，管理员可以查看。

### 系统设置（System Settings）
登录用户集中管理个人账户、会话、偏好和可用资源的统一入口；管理员在同一入口
额外管理用户、部门资源、配额、任务调度和审计，不使用另一套独立管理产品界面。
_避免使用：独立管理后台、服务器管理首页_

### 浏览器会话（Browser Session）
用户在一个浏览器或设备完成登录后形成的独立访问会话。用户可以识别当前
会话、查看其他有效会话，并单独退出其他会话或退出全部会话；改密、管理员
重置密码和账号停用会立即撤销相关会话。

### 运维 CLI（Operations CLI）
仅供服务器管理员执行账号初始化、密码恢复和数据维护的管理入口，不是用户
生图入口，不提供 Codex OAuth 能力。
_避免使用：本地 CLI、用户 CLI、生图 CLI_

### 个人图库（Personal Gallery）
每个用户私有的参考图素材库，上传素材默认进入这里。改造前的单用户版本中原名
"公用图库"——该旧名在多用户语境下有歧义（会被误解为跨用户共享），
故废弃："公用/共享"一词今后专指跨用户资源。其他普通用户不可查看，管理员
可以通过审计入口查看；管理员自己的个人图库仍只包含本人素材。

### 共享图库（Shared Gallery）
全部门可见的参考图素材库，存放品牌素材、产品图等公共资源。
由管理员统一创建和维护；普通用户可以浏览、筛选、预览和加入当前图像输入，
但不能新增、修改、替换、排序、分类或停用共享图库内容。
_避免使用：公用库、公共库、团队图库、团队管理库_

### 图库分类（Gallery Category）
组织图库参考图的分组。个人图库分类由各用户自行维护，共享图库分类由管理员
统一维护；两套分类相互独立，同名分类之间也没有关联。图库分类是一级分组，
不形成父子层级。共享图库始终保留不能删除的“未分类”，用于承接尚未整理的
素材和旧数据。

### 图库素材名称（Gallery Item Name）
图库中用于搜索和引用一项参考图的名称。同一图库内名称唯一；个人图库与共享
图库之间允许同名，展示和选择时必须明确标识所属图库。

### 图库素材（Gallery Item）
个人图库或共享图库中的一项参考图。每项素材必须有图片、所属图库内唯一的名称
和一个图库分类，可以附带说明引用方式的备注。

### 个人资产 / 共享资产（Personal / Shared Assets）
统一的归属规则：可复用资产——图库素材、提示词模板、提示词片段——
都分为个人层（各自私有）与共享层（全部门可读）；共享层的写权限由具体
资产类型定义。生成结果（任务与图片）只属于个人层，不进入共享层。个人层
对其他普通用户隔离，但管理员可以查看。

### 资产版本（Asset Version）
可复用资产的一次不可覆盖内容快照。替换文件或正文会创建新版本，旧版本
不再作为默认选择，但历史任务继续引用当时使用的版本。

### 个人回收站（Personal Trash）
用户删除的个人任务和图片进入的可恢复区域，图片文件默认保留 30 天后清理。
任务摘要、素材哈希、用量和审计记录不随图片文件清除。

### 用户存储额度（User Storage Quota）
管理员为单个用户设置的个人文件空间上限，个人图库、参考图、生成结果和
回收站文件都计入额度。超额后禁止新增文件，但不影响查看、下载和清理。

### 部门共享存储池（Shared Storage Pool）
共享图库、共享模板和共享提示词片段共同使用的部门文件空间，由管理员设置
总额度。共享文件不计入上传者个人额度，但保留上传者归属和审计记录。

### 个人供应商（Personal Provider）
用户从供应商目录中选择接口，并配置自己的 API Key 与允许使用的模型。
仅本人可用，费用与额度由本人承担；用户不能填写任意 Base URL。完整 API Key
不向管理员展示，管理员只能查看配置状态与用量，并可停用配置。

### 部门供应商（Shared Provider）
管理员从供应商目录中配置、全部门用户可选用的 API 供应商；API Key 对普通
用户不可见。适用于部门统一付费的账号，管理员设置全局及按用户的额度与
并发上限；超限时不得自动改用用户的个人供应商。

### 公平队列（Fair Queue）
按用户轮转选择下一项生成任务的部门任务队列，同时受用户并发、部门供应商
全局并发和额度约束，避免单个用户长期占满生成能力。

### 任务尝试（Task Attempt）
生成任务对供应商的一次实际调用。服务器重启时，等待任务继续排队，执行中
尝试标记为中断且不自动重跑；用户确认重试会创建新的任务尝试。

### 供应商目录（Provider Catalog）
管理员维护的可用 API 接口、调用方式和模型白名单。个人供应商和部门供应商
都必须从目录中选择，普通用户不能让服务器请求任意地址。

### 供应商版本（Provider Version）
供应商接口地址、调用方式和模型白名单的一次不可覆盖配置快照。任务记录
使用的版本和非敏感参数，但不保存 API Key 明文。

### 管理员（Administrator）
拥有管理权限的特殊用户：创建用户账号、重置任何用户的密码、停用账号，
统一创建和维护共享图库，
可在界面中查看所有用户的图片与任务历史（部门工具定位，内容属工作产出，
隔离的目的是"各自管理"而非"互相保密"），并可查看部门供应商的按用户
用量汇总。管理员访问用户内容必须写入审计日志，不能模拟用户登录、查看
个人 API Key 或使用个人供应商执行任务。

### 停用（Deactivation）
账号、资产和供应商退出可用状态的统一方式：数据、历史引用和审计记录原地
保留。管理员删除共享图库内容时实际执行停用，内容从普通用户视图中消失但
可以恢复；产品界面不提供物理删除，物理清理属于运维 CLI 操作。
